# Windows 작업 스케줄러 — 매일 오전 09:30 뉴스 수집 자동 실행

이 문서는 **비개발자**도 따라 할 수 있도록 작성했습니다.  
크롤러 코드는 바꾸지 않고, **`scripts/run_daily_crawl.bat`** 만 매일 실행하면 됩니다.

실제로 돌아가는 명령:

```text
python scripts/collect_news.py
```

---

## 자동 실행이 하는 일 / 하지 않는 일

| 하는 일 | 하지 않는 일 (이번 단계) |
|--------|-------------------------|
| RSS 뉴스 수집 | GitHub 자동 업로드 |
| `raw_data/날짜/` 원본 저장 | Vercel 자동 재배포 |
| `data/news.json` **로컬 PC에서** 갱신 | Supabase DB 저장 |
| `logs/daily_crawl_날짜.log` 기록 | LLM 요약 |

### 웹사이트(Vercel)에 반영하려면

- 작업 스케줄러는 **내 컴퓨터의** `data/news.json` 만 업데이트합니다.
- **인터넷에 올라간 Vercel 사이트** 내용을 바꾸려면, 지금은 **GitHub에 commit → push** 한 뒤 Vercel이 다시 배포되어야 합니다.
- **나중에 Supabase DB**를 연결하면, DB만 갱신해도 웹앱이 재배포 없이 최신 뉴스를 보여 줄 수 있습니다.

## A. 사전 준비 (한 번만)

### 1) Python 설치

1. https://www.python.org/downloads/ 에서 **Python 3.10 이상** 설치
2. 설치 화면에서 **「Add python.exe to PATH」** 반드시 체크
3. 설치 후 **새로** 명령 프롬프트를 열고 확인:

```text
python --version
```

버전이 나오면 성공입니다.

### 2) Python 패키지 설치

1. **파일 탐색기**로 프로젝트 폴더를 엽니다.  
   예: `C:\projects\bio-news-report`
2. 주소 표시줄에 `cmd` 입력 후 Enter → 해당 폴더에서 명령 창이 열립니다.
3. 아래 입력 후 Enter:

```text
pip install -r requirements.txt
```

오류 없이 끝나면 준비 완료입니다.

## B. 사전 테스트 (작업 스케줄러 등록 **전** 필수)

### 1) 테스트 배치 파일 더블클릭

1. 프로젝트 폴더 → **`scripts`** 폴더
2. **`test_daily_crawl.bat`** 더블클릭
3. 검은 창에 진행 메시지가 나옵니다. 끝날 때까지 기다립니다.
4. **`SUCCESS`** 가 보이면 성공, **`FAILED`** 면 아래 「J. 실패 시」 참고
5. 아무 키나 누르면 창이 닫힙니다.

### 2) 로그 확인

1. 프로젝트 폴더 → **`logs`** 폴더
2. **`daily_crawl_2026-06-04.log`** 처럼 **오늘 날짜** 파일이 생겼는지 확인
3. 메모장으로 열어 아래가 있는지 봅니다.
   - `[START]` 실행 시작 시간
   - `[PROJECT_ROOT]` 프로젝트 경로
   - `[PYTHON]` Python 경로
   - `[COMMAND]` 실행 명령
   - 크롤러 출력 내용
   - `[RESULT] SUCCESS` 또는 `FAILED`
   - `[END]` 종료 시간

### 3) 데이터 파일 확인

| 확인 위치 | 기대 결과 |
|-----------|-----------|
| `raw_data\오늘날짜\raw_items.json` | 새로 생기거나 수정 시간이 방금 |
| `raw_data\오늘날짜\deduplicated_items.json` |同上 |
| `data\news.json` | 수정 시간이 방금 |
| `data\news.backup.json` | 백업 파일 (있을 수 있음) |

### 4) 웹앱에서 확인 (선택)

```text
npm run dev
```

브라우저 → `/reports` → 오늘 날짜 리포트가 보이면 로컬 웹앱 반영 OK입니다.

## I. 전원 옵션 (노트북 사용자 필수에 가깝게)

09:30에 PC가 **절전/휴면**이면 작업이 안 돌 수 있습니다.

1. **제어판** → **전원 옵션** (또는 설정 → 시스템 → 전원)
2. **고급 전원 설정 변경** → 현재 사용 중인 **전원 관리 옵션 변경**
3. **절전** → **절전 모드 해제 타이머 허용** → **사용** (배터리·전원 연결 모두 확인)

또는 **09:30 전후에 PC가 켜져 있고** 화면이 꺼져도 절전만 안 되게 설정하는 것이 안전합니다.

## K. 실패 시 확인할 것

| 순서 | 확인 |
|------|------|
| 1 | `logs` 폴더 **가장 최근** `daily_crawl_*.log` 열기 |
| 2 | `[RESULT] FAILED`, `[EXIT_CODE]` 줄 확인 |
| 3 | Python 설치·PATH (`python --version` in cmd) |
| 4 | `pip install -r requirements.txt` 다시 실행 |
| 5 | 프로젝트 폴더를 **이동**했으면 작업 스케줄러 **동작 탭 경로** 수정 |
| 6 | **인터넷** 연결 |
| 7 | Windows **로그인 비밀번호** 변경 여부 → 작업 속성에서 비밀번호 재입력 |
| 8 | 09:30에 PC **절전/꺼짐** 여부 → 전원·해제 타이머 설정 |
| 9 | `test_daily_crawl.bat` 더블클릭으로 **동일 오류** 재현되는지 확인 |

로그에 `Python NOT FOUND` → Python PATH 문제.  
`ModuleNotFoundError: feedparser` → `pip install -r requirements.txt` 미실행.

## M. 자주 묻는 질문

**Q. 09:30에 자동으로 돌았는지 어떻게 알아요?**  
→ `logs` 폴더에 그날짜 로그가 있고 `[START]` 시간이 09:30대인지 보면 됩니다.

**Q. Vercel 사이트는 왜 안 바뀌나요?**  
→ 자동 실행은 **이 PC의 파일**만 바꿉니다. GitHub push + Vercel 배포가 필요합니다 (Supabase 연결 후에는 DB만 갱신 가능).

**Q. 크롤러를 손으로 돌리고 싶어요.**  
→ `test_daily_crawl.bat` 더블클릭 (결과 확인 후 창 닫기).
